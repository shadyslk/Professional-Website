Professional Website

Vision

The idea for this website came from wanting to take the knowledge of the technologies that I've learned and showcase them. I wanted to build a website that I could use to present the projects that I have created.

Technologies Used

  • Amazon S3
  • Amazon Route 53
  • Amazon CloudFront
  • Amazon Certificate Manager
  • HTML
  • CSS

Process

Design

The two longest pieces of this project were design and content. Design wise, I had decided early on that I wanted to keep the website simple. I am a huge fan of a variety of designs. There are plenty of great examples of beautiful designs that are very complex and admirable. I have a philosophy of keeping things simple. In web design, this can result in faster load times overall and provides a clear message.

CSS and HTML have come a long way over the years and provide a lot of out-of-the-box functionality that helps with my design philosophy. CSS now can handle animations and transitions beautifully. I got the chance to take advantage of these in the creation of the website.

Once the overall design was created, it became easier to fill in the content as I had started making my outline in the process. I have learned from experience that it great to start broad and then become more specific as needed.

I have linked my GitHub below where my code for this project is linked for reference. It should update regularly as this site will update.

Hosting

We are in an age in web development where there are lots of options for hosting a website. GitHub offers static website hosting as a main example. I could have also gone a slightly more traditional route and spun up a virtual machine to host the website. In the end, I chose Amazon S3 for hosting this static website.

The choice of Amazon S3 was one that worked well for my situation. It is cost effective because I don't expect a heavy amount of traffic at this time and S3 will only charge me for what is used. It is also what I am comfortable with as I am an AWS Certified Solutions Architect and integrates well with other technologies I am familiar with.

Security

The final piece I had to establish was ensuring that my website had a TLS certificate to ensure that connections to my website were secure.

I had generated my TLS certificate using Amazon Certificate Manager. I also took the time to set up a CloudFront Distribution. This would allow me to attach my TLS Certificate to my static website. It also has the benefit of distributing my website to users more efficiently.

Lastly, I simply had to create a new Domain and DNS Entry to point to my CloudFront Distribution. This way, I have a professional looking domain name to share with users.

Conclusions

Overall, I have found the process to be very educational in having me practice my craft. I feel this is a great first step to showcasing my talents. I got to create a website using HTML and CSS for design and content. I also worked with Amazon Web Services to host my website and ensure it is secure.
